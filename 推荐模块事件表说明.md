# 推荐模块事件表

这份表用于给“模块表现总览”里的单模块趋势图提供事件注释层。建议将表格作为独立数据源维护，不要和指标明细表混在一起。

## 使用建议

- 优先使用 [推荐模块事件表.csv](/Users/miao/Desktop/推荐看盘自由发挥版/推荐模块事件表.csv) 作为录入模板。该文件可直接用 Excel 打开和维护。
- 每条记录代表一个“需要在趋势图上被看见的动作”。
- 点事件和阶段事件都用同一张表维护。
- `模块名称` 可填写具体模块，也可填写 `ALL` 表示全局动作。
- 页面展示时建议只上图 `high` 和部分 `medium` 级别事件，避免趋势图过于拥挤。

## 字段定义

| 字段 | 是否必填 | 示例 | 说明 |
| --- | --- | --- | --- |
| `事件ID` | 是 | `EVT-202601-001` | 事件唯一标识，建议按 `EVT-月份-序号` 编码。 |
| `模块名称` | 是 | `商详推荐` / `ALL` | 对应趋势图的模块名；`ALL` 表示作用于所有推荐模块。 |
| `适用范围` | 是 | `单模块` / `全局` | 辅助说明该事件是模块级还是全局级。 |
| `事件类型` | 是 | `launch` | 建议枚举：`launch`、`gray`、`experiment`、`rollback`、`strategy_change`。 |
| `事件级别` | 是 | `high` | 建议枚举：`high`、`medium`、`low`，用于控制图上展示优先级。 |
| `阶段类型` | 是 | `点事件` | 建议枚举：`点事件`、`阶段事件`。 |
| `开始月份` | 是 | `202601` | 统一使用 `YYYYMM`。趋势图按月聚合时，以此字段做定位。 |
| `结束月份` | 否 | `202603` | 阶段事件必填；点事件可与开始月份相同或留空。 |
| `精确日期` | 否 | `2026-01-10` | 用于 tooltip 展示真实发生日期，图上仍按月展示。 |
| `事件标题` | 是 | `商详推荐新版召回策略全量上线` | 趋势图上显示的短标题。建议控制在 12-20 个字。 |
| `事件摘要` | 否 | `替换旧版召回策略并提升高相关商品覆盖率` | tooltip 或侧边详情中的说明。 |
| `灰度进度` | 否 | `10%->30%->100%` | 灰度或实验阶段建议填写，帮助解释“不是一个时间点”的事件。 |
| `影响指标` | 否 | `曝光UV|UV价值` | 建议用 `|` 分隔，支持多个指标。 |
| `预期方向` | 否 | `提升` | 建议枚举：`提升`、`下降`、`待验证`、`中性`。 |
| `状态` | 是 | `已全量` | 建议枚举：`规划中`、`灰度中`、`已生效`、`已全量`、`已结束`、`已回滚`。 |
| `负责人` | 否 | `张三` | 便于回溯和补充复盘信息。 |
| `需求/实验ID` | 否 | `REQ-1024` | 可关联需求单、实验单、发布单。 |
| `关联文档` | 否 | `https://example.com/doc/REQ-1024` | 页面可作为外链跳转。 |
| `备注` | 否 | `按月图中展示为 202601 的上线标记` | 存放特殊说明。 |

## 推荐录入规范

### 1. 点事件

适用于正式上线、回滚、策略切换等单点动作。

- `阶段类型` 填 `点事件`
- `开始月份` 填动作所属月份
- `结束月份` 可留空，或与 `开始月份` 相同
- `精确日期` 尽量填写，方便 tooltip 展示

页面表达建议：

- 在当月的趋势点上叠加一个小标记
- tooltip 中显示精确日期和事件标题

### 2. 阶段事件

适用于灰度、AB 实验、渐进放量等持续一段时间的动作。

- `阶段类型` 填 `阶段事件`
- `开始月份`、`结束月份` 都必填
- `灰度进度` 尽量填写，比如 `10%->30%->100%`
- 如果事件仍在进行，`结束月份` 可先填当前月并持续更新

页面表达建议：

- 在趋势图上用跨月份的浅色背景带表示事件持续区间
- 在区间起点或中部展示简短标签
- tooltip 中展示精确日期、阶段范围和灰度进度

## 推荐展示规则

- `high`：默认上图
- `medium`：同月事件不多时上图，否则只进 tooltip
- `low`：默认不在图上显式打点，只在事件列表或 tooltip 中展示

同一个月存在多条事件时：

- 图上只显示 `本月 3 个动作` 这类聚合标签
- 悬停后展开事件清单

## 维护流程建议

1. 每次灰度、上线、回滚后由对应 owner 补录事件表。
2. 月报复盘前补齐 `事件摘要`、`预期方向`、`关联文档`。
3. 页面读取时先按 `模块名称` 精确匹配，再补充 `ALL` 级全局动作。
4. 若趋势图仍按月展示，不要在图上制造“日级精度”的错觉；精确日期只放到 tooltip 和详情里。
