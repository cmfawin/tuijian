# 优鲜 APP 推荐体系经营看板 PRD

## 1. 文档信息

- 文档版本：V1.1
- 文档状态：已按当前看板实现对齐
- 更新时间：2026-03-24
- 主数据源：`1321283_优鲜流量千人千面月报_交叉表_20260321_1135.xlsx`
- 事件数据源：`推荐事件表.xlsx`

## 2. 项目背景

当前优鲜 APP 推荐相关数据已经沉淀到月度交叉表，但仍需要一张能够同时服务管理层和运营分析同学的统一看盘页面。

本看板的目标不是做一个“全能 BI”，而是形成一条固定阅读链路，让用户能在一页内完成：

- 看推荐体系本月整体表现
- 看结构化经营摘要
- 看推荐加权 UV 价值波动归因
- 看模块机会矩阵
- 看模块明细和单模块诊断

## 3. 产品目标

### 3.1 业务目标

- 让管理层在 1 分钟内看懂推荐体系本月整体状态和重点变化
- 让运营分析同学在 5 分钟内定位拖累模块、识别研发优先级，并下钻到单模块趋势

### 3.2 产品目标

- 提供固定阅读顺序：`核心指标 -> 本月经营摘要 -> 趋势 -> 波动归因 -> 优先级矩阵 -> 模块总览 -> 单模块诊断`
- 支持按月份统一联动整页分析
- 支持把已维护的事件信息叠加到单模块趋势诊断中，帮助判断动作和指标变化的关系

## 4. 数据范围与口径

### 4.1 时间范围

- 当前主数据可用月份：`202503` 至 `202602`
- 趋势图统一按 `近 12 个月` 展示
- 页面默认选中最新月份

### 4.2 分析对象

当前页面围绕以下对象展开：

- 19 个推荐模块
- 1 个全站参照
- 1 个推荐加权分析口径

页面不直接展示：

- `搜索结果`
- `推荐加权` 原始行

说明：

- 名称里带“搜索”的若干模块，仍属于推荐模块，保留在推荐体系分析范围内

### 4.3 主数据字段

当前主数据源字段为：

- 月份
- 模块
- 曝光UV(udid)
- 支付转化率
- UV(udid)价值
- 引导成交业绩

### 4.4 事件数据字段

当前事件表已接入字段为：

- 事件ID
- 模块名称
- 事件类型
- 阶段类型
- 开始月份
- 结束月份
- 精确开始日期
- 精确结束日期
- 事件标题
- 事件内容
- 灰度方案（仅 grey 事件适用）

### 4.5 当前数据限制

当前 V1 不包含：

- 主动流量口径
- 商品 PV 点击率
- 日级主趋势
- 自由问答式助手
- 运行时直接读取 `.xlsx`

说明：

- 当前事件表是离线整理后接入页面，不是浏览器运行时直接解析 Excel

## 5. 页面定位与信息架构

### 5.1 页面定位

页面定位为 `桌面经营分析页`，兼顾汇报展示与专项诊断，但优先保证“经营判断清晰”和“分析路径顺畅”。

### 5.2 页面结构

页面采用单页滚动布局，顺序固定如下：

1. 页面标题与月份筛选
2. 核心指标卡
3. 本月经营摘要
4. 核心指标趋势图
5. 推荐加权 UV 价值月环比波动归因
6. 算法研发优先级矩阵
7. 模块表现总览
8. 单模块表现诊断

## 6. 全局筛选器

### 6.1 筛选项

- 月份选择

### 6.2 默认值

- 默认选中主数据最新月份

### 6.3 联动规则

- 用户切换月份后，核心指标、经营摘要、趋势图、波动归因、优先级矩阵、模块总览、单模块诊断统一刷新
- `本月经营摘要` 标题中的月份标签同步显示当前筛选月份
- 若切到最早月份，所有环比相关字段显示为“无上月数据”或中性状态，不报错

## 7. 功能模块详细需求

## 7.1 核心指标卡

### 7.1.1 指标数量

共 4 张卡：

1. 推荐模块数
2. 推荐加权 UV 价值
3. 推荐加权支付转化率
4. 全站 UV 价值

### 7.1.2 指标定义

- 推荐模块数：当前纳入看盘的推荐模块数量，当前为 19
- 推荐加权 UV 价值：按模块引导成交业绩占比加权后的模块 UV 价值
- 推荐加权支付转化率：按模块引导成交业绩占比加权后的模块支付转化率
- 全站 UV 价值：全站当月 UV 价值

### 7.1.3 展示内容

每张卡展示：

- 指标名称
- 当月绝对值
- 环比描述
- 趋势箭头
- 装饰图标

### 7.1.4 颜色规则

- 环比上升：红色
- 环比下降：绿色
- 无环比数据：灰色

### 7.1.5 交互规则

- 当前实现中，第 1 张“推荐模块数”卡片不支持点击展开趋势
- 第 2、3、4 张卡片支持点击切换趋势图
- 默认选中第 2 张卡：`推荐加权 UV 价值`
- 再次点击当前选中卡片，则折叠趋势图

## 7.2 本月经营摘要

### 7.2.1 目标

在用户看完核心指标后，立刻给出结构化经营判断，降低人工归纳成本。

### 7.2.2 展示位置

位于核心指标卡下方、趋势图上方。

### 7.2.3 标题规则

- 标题格式为：`本月经营摘要（XX年X月）`
- 括号中的月份来自当前筛选月份

### 7.2.4 输出结构

固定输出 3 行：

1. 整体判断
2. 主要变化
3. 下一步动作

### 7.2.5 输出要求

- 整体判断：1 段，概括推荐体系本月整体表现
- 主要变化：1-2 条，必须点名具体模块或关键变化来源
- 下一步动作：1-2 条，必须是可执行动作

### 7.2.6 口径来源

当前摘要基于以下页面内数据计算，不依赖大模型对话：

- 推荐加权 UV 价值
- 上月对比结果
- 波动归因 Top 模块
- 优先级矩阵中的“优先优化/放量验证”结果

## 7.3 核心指标趋势图

### 7.3.1 目标

在用户切换顶部核心指标卡后，展示对应指标近 12 个月趋势。

### 7.3.2 展示规则

- 默认展开
- 默认展示：`推荐加权 UV 价值`
- 一次只展示 1 个主图

### 7.3.3 图表规则

- X 轴：月份
- 时间窗口：近 12 个月
- 卡 2：推荐加权 UV 价值主线 + 全站 UV 价值参照线
- 卡 3：推荐加权支付转化率主线 + 全站支付转化率参照线
- 卡 4：全站 UV 价值单线

### 7.3.4 Tooltip 规则

鼠标悬停时展示：

- 月份
- 当前系列值
- 若存在参照线，则同时展示参照值

## 7.4 推荐加权 UV 价值月环比波动归因

### 7.4.1 目标

将推荐加权 UV 价值的月环比变化拆解到具体模块，明确“谁在拉升，谁在拖累”。

### 7.4.2 当前实现边界

- 当前实现固定展示 `推荐加权 UV 价值` 的归因
- 不支持切换为推荐加权支付转化率归因
- 该模块在页面中固定展示，不跟随卡片显隐联动

### 7.4.3 展示结构

模块包含 3 部分：

1. 顶部 3 张摘要卡
2. 口径解释说明框
3. 归因拆解表

### 7.4.4 摘要卡字段

1. 综合变化
2. UV价值贡献
3. 权重贡献

### 7.4.5 归因表字段

- 模块名称
- 当前月引导成交业绩
- 上月权重 -> 本月权重
- 权重变化
- 上月 UV 价值 -> 本月 UV 价值
- UV 价值变化
- 综合贡献
- 权重贡献
- UV 价值贡献
- 归因判断

### 7.4.6 排序规则

- 按综合贡献绝对值降序

### 7.4.7 当前口径公式

定义：

- 模块权重 = 该模块当月引导成交业绩 / 全部推荐模块当月引导成交业绩之和
- 推荐加权 UV 价值 = 所有推荐模块的 `模块权重 × 模块 UV 价值` 求和

拆解口径：

- `UV价值贡献 = ((本月权重 + 上月权重) / 2) × (本月UV价值 - 上月UV价值)`
- `权重贡献 = ((本月UV价值 + 上月UV价值) / 2) × (本月权重 - 上月权重)`
- `综合贡献 = UV价值贡献 + 权重贡献`

## 7.5 算法研发优先级矩阵

### 7.5.1 名称

模块名称统一为：`算法研发优先级矩阵`

### 7.5.2 目标

在研发资源有限时，快速识别最值得优先投入的模块。

### 7.5.3 展示对象

- 仅展示 19 个推荐模块

### 7.5.4 坐标定义

- X 轴：模块业绩权重
- Y 轴：相对推荐加权 UV 价值均值的偏离
- 点大小：引导成交业绩
- 点颜色：动作建议类型

### 7.5.5 分割线规则

- 纵向分割线：模块业绩权重中位数
- 横向分割线：`0`，即是否高于/低于推荐盘均值

### 7.5.6 输出内容

页面除矩阵图外，还包括：

- 权重分界摘要卡
- 优先优化摘要卡
- 放量验证摘要卡
- 右侧“优先优化”模块列表
- 右侧“放量验证”模块列表
- 使用说明卡

### 7.5.7 动作分类

- 优先优化：高权重且低于盘均 UV 价值
- 放量验证：低权重但高于盘均 UV 价值
- 核心守盘：高权重且高于盘均
- 观察收缩：低权重且低于盘均

## 7.6 模块表现总览

### 7.6.1 目标

从全量模块视角查看本月表现，并作为单模块诊断的唯一下钻入口。

### 7.6.2 展示对象

- 19 个推荐模块
- 1 行全站参照

不展示：

- 推荐加权
- 搜索结果

### 7.6.3 展示字段

- 模块名称
- 曝光UV
- 支付转化率
- UV 价值
- 引导成交业绩

### 7.6.4 默认排序

- 按引导成交业绩降序
- 全站行固定置底

### 7.6.5 排序能力

支持按以下字段排序：

- 曝光UV
- 支付转化率
- UV 价值
- 引导成交业绩

### 7.6.6 交互规则

- 点击任意推荐模块行，在表格内展开单模块诊断
- 点击全站行不触发单模块诊断
- 页面展示导出按钮样式，但当前未接出实际导出逻辑

## 7.7 单模块趋势与诊断

### 7.7.1 目标

在总览之后，对单个推荐模块进行近 12 个月趋势与动作诊断。

### 7.7.2 触发方式

- 点击模块表现总览中的任意推荐模块行后，在该行下方展开

### 7.7.3 展示内容

展开区域包含：

- 模块标题
- 指标切换器
- 关键动作注释
- 单模块趋势图

### 7.7.4 指标切换项

当前实现仅支持以下 3 种切换：

1. `曝光/UV价值`
2. `支付转化率`
3. `引导成交业绩`

说明：

- `曝光/UV价值` 为双轴图：曝光UV + UV价值
- 不再单独提供“UV价值”单选 Tab
- `商品PV点击率` 当前数据源不支持，不展示

### 7.7.5 图表规则

- 默认展示近 12 个月
- 单选切换指标
- `曝光/UV价值` 使用双轴折线
- 其他指标使用单轴折线
- Tooltip 展示月份和对应数值

### 7.7.6 事件注释能力

当前已支持将事件表中的已匹配模块叠加到单模块诊断中。

表现方式包括：

- 图上方“关键动作注释”摘要卡
- 图内灰度/回退遮罩
- 图内上线竖线
- Tooltip 中追加当月动作信息

当前已接入的模块包括：

- 首页feeds
- 购物车底部推荐
- 结算页顺手买

当前未匹配事件数据的模块：

- 展示“暂无已接入的事件标注”

## 8. 视觉与体验要求

### 8.1 页面体验

- 页面宽屏优先，适配主流 PC 分辨率
- 首屏需在不滚动情况下看到标题、筛选器、核心指标和经营摘要
- 所有图表与表格需和月份筛选联动

### 8.2 易读性

- 指标名称统一，不出现“访问UV/曝光UV”混用
- 模块名称以当前主数据字段为准
- 强调色统一使用顶部品牌红 `primary`

### 8.3 性能

- 当前数据量较小，前端一次性加载即可
- 切换月份时优先保证图表稳定刷新
- 事件数据当前通过静态文件接入，满足当前体量
